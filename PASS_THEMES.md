# Collectible pass themes

EventPass now renders attendee credentials through one normalized data model and one shared interaction engine. Seven visual presets provide theme-specific composition, decoration, typography, surface treatment, and animation without changing pass validation or QR contents.

## Architecture

- `pass-system.ts` defines the normalized model, theme IDs, registry, and safe customization helpers.
- `pass-data.ts` maps the existing Prisma event, attendee, pass, QR, and branding records into that model.
- `collectible-pass.tsx` owns pointer tilt, flip state, keyboard interaction, visibility pausing, QR rendering, and optional-field behavior.
- `pass-theme-art.tsx` contains lightweight CSS/SVG artwork for the seven themes.
- `collectible-pass.module.css` contains shared design tokens plus the distinct Casino, Gala, Retro Arcade, Science, Biology, Space, and Minimal art systems.
- `/dashboard/pass-designs` is the development and organizer preview surface.

## Existing field mapping

| Existing source | Normalized field | Presentation |
| --- | --- | --- |
| `Event.name` | `eventName` | Primary event title |
| `Event.description` | `customMessage` | Optional reverse-side message |
| `Event.startsAt`, `Event.endsAt` | `eventDate`, `eventTime` | Localized credential metadata |
| `Event.venue`, `Event.address` | `venueName`, `venueLocation` | Venue or destination details |
| `Event.photoUrl`, `bannerImageUrl` | `backgroundImage` | Optional low-opacity artwork beneath content |
| `Event.organizer`, `Event.logoUrl` | `organizerName`, `organizerLogo` | Organizer mark with branding fallback |
| `Event.passTheme` | renderer `theme` | Event-wide visual design selected by the organizer |
| Attendee first and last name | `attendeeName` | Credential holder identity |
| `Attendee.company` | `company` | Optional identity subtitle |
| `Attendee.ticketTier`, `vip` | `passType`, `accessLevel` | Pass badge and access metadata |
| `Attendee.raffleTickets` | `raffleTickets` | Theme-specific ticket/token/credit count |
| `Attendee.seat` | `gate` | Optional seat or gate metadata |
| `Attendee.status` | `status` | Front status badge |
| Plus-one fields | `companionName` | Optional reverse-side companion detail |
| Allergen fields | `advisories` | Optional reverse-side check-in advisory |
| `Pass.id`, `fallbackCode` | `passId`, `fallbackCode` | Unique and fallback credentials |
| Existing signed QR payload | `qrDataUrl` | Generated by the existing QR service and isolated from visual effects |
| Organization colors | `accentColor`, `secondaryColor` | Sanitized CSS design tokens |

The current database does not provide attendee photos, sponsor-logo collections, or a dedicated perks collection. Those normalized fields remain optional and collapse without empty labels. They can be populated later without changing the renderer.

## Usage

```tsx
<CollectiblePass data={normalizedPass} theme="casino" />
```

Supported theme values are `casino`, `gala`, `retro-arcade`, `science`, `biology`, `space`, and `minimal`. The organizer selects the design while creating an event; it is stored in `Event.passTheme`, inherited by every attendee pass, preserved when an event is duplicated, and can be corrected from the event card later. Attendee-facing pages intentionally do not expose their own theme selector. Minimal supports `genericVariant="light"` and `genericVariant="dark"`.

The renderer accepts controlled or uncontrolled front/back state, `forceReducedMotion`, `animationIntensity`, compact gallery sizing, safe accent colors, organizer assets, and optional background artwork.

## Accessibility and performance

- Enter and Space flip the focused pass; visible controls also flip either face.
- Hidden faces are both `aria-hidden` and inert, preventing hidden controls from receiving focus.
- Pointer updates use a cached bounding rectangle and `requestAnimationFrame`.
- Pointer exit and flipped state reset tilt.
- `prefers-reduced-motion` and the explicit reduced-motion preview disable nonessential movement.
- An intersection observer pauses CSS animation when a pass leaves the viewport.
- Decorative artwork is CSS/SVG, hidden from assistive technology, and kept below the protected white QR surface.
- Long titles and names clamp safely; absent optional fields do not reserve space.

The existing SVG download remains available as the stable printable fallback; the new interactive themes are the on-screen digital credential. Calendar links, QR payload disclosure, signed scanning, fallback validation, and pass email URLs remain unchanged.
